# 변경 이력

이 프로젝트의 모든 주요 변경사항은 이 파일에 기록됩니다.

형식은 [Keep a Changelog](https://keepachangelog.com/ko/1.0.0/)를 따르며,
이 프로젝트는 [Semantic Versioning](https://semver.org/lang/ko/)을 준수합니다.

이 저장소는 AI 에이전트 도구·스택을 한눈에 비교하는 큐레이션 카탈로그 사이트
(awesome-ai-stack)입니다. Astro로 만든 정적 사이트이며 도구 카탈로그, 블로그,
실행 가능한 예제(`samples/`)를 한국어·영어로 제공하고 GitHub Pages로 배포됩니다.
1.0.0은 카탈로그·상세 페이지·예제·다국어를 갖춘 첫 정식 릴리스를 뜻합니다.

## 1.10.0 - 2026-07-20

이번 릴리스의 중심은 **네이티브 프레젠테이션 슬라이드**입니다. MDX 파일 하나가
곧 발표 덱이 되는 기능을 새로 만들어 `/slides/` 허브와 덱별 페이지에서 보여주며,
분할·인용·통계·비교 레이아웃, 시각 테마, 떠 있는 목차, PDF 인쇄, 확대 라이트박스,
스크롤 연동 워크스루까지 발표에 필요한 요소를 갖췄습니다. 여기에 본문 내부 링크의
위키링크 전환과 내부 탐색을 같은 탭에서 여는 수정을 함께 담았습니다.

### 추가됨 1.10.0

- **네이티브 프레젠테이션 슬라이드** — MDX로 쓰는 슬라이드 덱과 이를 모아 보는
  허브(`/slides/`)·덱별 페이지(`/slides/<deck>/`)를 추가. 한국어·영어 로케일 분리,
  덱별 이미지 폴더, 표시 순서·초안 프런트매터를 지원
- **슬라이드 레이아웃** — 코드·다이어그램·이미지를 나란히 놓는 분할(`aas-split`)과
  이미지 fit·세로 정렬 모드, 인용·통계·콜아웃·비교 레이아웃, 마크다운 컬럼,
  상단 정렬 제목과 `::sub` 부제, 자동 맞춤(auto-fit)
- **덱 테마와 시각 스타일** — 덱 단위 기본 테마·기본 슬라이드 타입 지정과
  드로어 안의 시각 스타일 전환기
- **탐색·발표 도구** — 좌측 가장자리 액션 드로어, 슬라이드 제목만 모은 떠 있는
  목차, 전환 시간 제어, 화면비·단어 줄바꿈 설정, 푸터, PDF 인쇄
- **미디어 확대와 대형 다이어그램** — 이미지·다이어그램 클릭 확대 라이트박스,
  가로로 스크롤되는 대형 다이어그램(`scroll-x`), 분할 안에서 스크롤되는 다이어그램
- **스크롤 연동 워크스루** — `:::step` 지시어가 코드·이미지·다이어그램의 스크롤
  위치를 끌고 가는 단계별 워크스루. 코드는 단계마다 전체 줄 하이라이트를 지원
- **코드 슬라이드 표시** — 코드 언어 배지, 슬라이드별 출처 표기, 큰 모노스페이스 서체
- **레이아웃·기능 샘플 덱** — 슬라이드 기능 전반을 훑는 한국어·영어 샘플 덱

### 변경됨 1.10.0

- **내부 링크 위키링크 전환** — 본문의 개념·도구·글 내부 링크를 `[[위키링크]]`
  표기로 통일하고, 표 셀 안 위키링크의 파이프 이스케이프 등 표기를 정리
- **내부 링크를 같은 탭에서 열기** — 사이트 내부 탐색은 현재 탭에서 열도록 바꾸고,
  이제 필요 없어진 모바일 전용 새 탭 정책 스크립트를 제거

### 수정됨 1.10.0

- **슬라이드 전환 안정화** — 빠르게 넘길 때 애니메이션 슬라이드가 뒤로 튀던 문제와
  전환 지터를 고치고 전환 시간을 다듬음
- **인쇄/PDF 보정** — 분할 레이아웃을 유지한 채 인쇄되고, 끝에 붙던 빈 페이지를 제거
- **라이트박스 표시** — 배경을 불투명하게 해 뒤 슬라이드가 비치던 문제를 없애고,
  확대한 다이어그램의 라벨이 중심을 벗어나던 문제를 서체·transform 확대로 고침
- **로컬 샘플 실행기(개발 전용)** — Docker-out-of-Docker가 아닐 때도 모든 샘플이
  분리 실행되고 로그가 남도록 고침

## 1.9.1 - 2026-07-09

이번 릴리스의 중심은 **프로젝트 리브랜딩**입니다. 이름을 `awesome-agent-stack`에서
`awesome-ai-stack`으로 바꿔, 다루는 범위를 에이전트 너머 AI 전반(모델·서빙·RAG·
MLOps)으로 넓혔습니다. 여기에 문서 내 섹션 위키링크 `[[#앵커]]`를 새로 지원하고,
개념 문서의 문구와 한국어 용어·줄바꿈을 함께 손질했습니다.

### 추가됨 1.9.1

- **문서 내 섹션 위키링크** — `[[#앵커]]`·`[[#앵커|텍스트]]`로 같은 문서의 heading에
  링크하고, 텍스트를 생략하면 그 heading의 제목을 그대로 가져와 표시. 명시 `{#id}`가
  없는 앵커는 빌드에서 걸러짐

### 변경됨 1.9.1

- **프로젝트 이름 변경: `awesome-agent-stack` → `awesome-ai-stack`** — 다루는 범위를
  에이전트 너머 AI 전반(모델·서빙·RAG·MLOps)으로 넓히기 위한 리브랜딩. 설정·페이지
  타이틀·URL·User-Agent·README에 반영
- **소개 문구를 AI 시스템으로 확장** — 사이트·개념·블로그 태그라인과 README를
  "AI 에이전트"에서 "AI 시스템"으로 넓히고 서빙을 범위에 추가
- **카테고리 서술 중립화** — LLM 제공자·벡터 스토어·관측·가드레일 카테고리의 상위
  설명을 에이전트 종속 표현에서 AI 전반으로 다시 씀
- **한국어 용어 정리: 부작용 → 부수효과** — side effect의 표준 표기에 맞춰 세 개념
  문서에서 교체
- **개념 문서 문구 손질** — harness-engineering·agent-tools에서 em-dash로 이어지던
  절을 문장으로 나누고, "왜 중요한가"·"오케스트레이션" 상호참조를 앵커 링크로 바꿈

### 수정됨 1.9.1

- **한국어 줄바꿈** — 개념 본문에 `word-break: keep-all`을 적용해 한국어가 글자
  단위가 아니라 단어(어절) 단위로 줄바꿈되도록 고침

## 1.9.0 - 2026-07-08

이번 릴리스의 중심은 **샘플을 일급 시민으로 끌어올린 것**입니다. 실행 가능한
예제마다 전용 페이지(`/sample/<folder>/`)와 이를 모아 보는 허브(`/sample/`)를 새로
만들고, 허브를 검색·사용 도구 필터·정렬·페이징을 갖춘 목록으로 다시 설계했습니다.
여기에 `[[용어]]` 위키링크로 엮이는 **용어집**, LiteLLM과 LangGraph만으로 직접
구성한 코드 샌드박스 에이전트 예제, 그리고 로컬 Node 없이 Docker로 사이트를 돌리는
**기여자 개발 환경**(Docker Compose·로컬 실행 마법사·한국어 README)을 함께 담았고,
전체 코드 리뷰에서 나온 지적 18건을 정리했습니다.

### 추가됨 1.9.0

- **용어집** — 본문의 `[[용어]]` 위키링크가 중앙 링크로 해석되고, 모든 용어를 모아
  보는 용어집 페이지를 추가. 검색, 도구·개념 분리, 카테고리 그룹, 아이콘을 갖추고,
  한국어·영어 어느 쪽으로 써도 조회되며, 외부 링크·정의 전용 용어와
  `[[용어#앵커|텍스트]]` 섹션 링크, `[[슬러그|텍스트]]` 글 링크까지 지원
- **샘플 페이지 일급 지원** — 예제마다 전용 페이지(`/sample/<folder>/`)와 이를 모으는
  허브(`/sample/`)를 추가. README 목차와 파일 목록을 담은 고정 우측 레일, 읽기 서체,
  이 샘플을 쓰는 글·스택으로 가는 "이 샘플을 쓰는 곳" 백링크를 함께 실음
- **샘플 인덱스 검색·필터·정렬** — 허브를 세로 1열 목록으로 다시 짜고, 검색(제목·내용·
  제목+내용 범위 선택), 사용 도구 다중 선택 필터(선택한 도구를 모두 쓰는 샘플만, AND),
  정렬(최신·이름순), 페이지당 5개 페이징을 추가. 상태는 URL(`?q`·`?scope`·`?tools`·
  `?sort`·`?page`)에 담겨 공유·복원되고, 페이저는 그룹 단위 이동과 처음·끝 버튼을 갖춤
- **직접 구성한 코드 샌드박스 에이전트** — LiteLLM과 LangGraph만으로(LangChain 없이)
  같은 에이전트를 손수 조립한 실행 가능한 샘플(`samples/docker_2`)과 글을 추가하고,
  "직접 구성 vs `create_agent`"를 나란히 비교하는 글, 히어로 이미지, 전체 구조/세부
  구조 두 단계로 코드를 읽는 레이아웃을 함께 실음
- **LangChain 카탈로그 등록** — LangChain을 오픈소스 도구로 카탈로그에 추가하고 로고를 실음
- **로컬 샘플 실행 마법사(개발 전용)** — 개발 서버의 `/local-samples`에서 예제의 `.env`를
  편집하고, 작업을 입력하고, Docker 이미지 빌드부터 실행(Docker-out-of-Docker)까지
  브라우저에서 진행. 실행 레시피 선택, 새로고침을 넘어 이어지는 실행 기록·경과 시간,
  로그 폴링을 포함
- **Docker Compose 개발 환경과 한국어 README** — 로컬 Node 설치 없이 `docker compose`로
  사이트를 띄우도록 하고, README 시작하기를 Docker 중심으로 정리. 한국어 README를
  `docs/readme/README.ko.md`로 추가하고 문서 상단에 언어 선택을 넣음
- **릴리스 체크리스트 문서** — 새 버전을 낼 때 갱신·수행할 항목을 정리한 `docs/for-release.md`를 추가

### 변경됨 1.9.0

- **README 전면 재작성** — 현재 라우트·콘텐츠 컬렉션·기능에 맞게 README를 다시 씀
- **한국어 용어 정리** — 어색하던 "배선"을 문맥에 따라 "직접 구성·조립·연결 코드" 등으로 교체
- **글 임베드 샘플 표시** — 글 본문의 `SampleProject`가 뷰어 전체를 펼치는 대신 관련 샘플로
  가는 콜아웃을 렌더링하도록 바꾸고, 샘플 페이지에서는 관련 도구 카드를 감춤(그 연결은
  샘플이 아니라 참조하는 글·스택의 프런트매터에 있으므로)

### 수정됨 1.9.0

- **코드 리뷰 지적 18건 정리** — 전체 코드 리뷰에서 나온 HIGH 2건·MEDIUM 4건·LOW 12건을
  모두 해결. 그중 GitHub 릴리스 정보 캐시가 레이트리밋으로 일부만 받아올 때 캐시된 버전을
  덮어써 버리던 문제를 함께 고침
- **샘플 인덱스 레이아웃 안정화** — 좁은 폭에서 카드 그리드가 컨테이너를 넘어 설명이 잘리지
  않던 문제, 마지막 페이지·검색 결과가 적을 때 페이저 위치와 목록 높이가 흔들리던 문제를 해결
- **모바일 링크 동작 보정** — 좁은 화면에서 내부 링크만 현재 창으로 열고, 외부 링크는 새 탭을 유지하도록 함
- **코드 샘플 선택기 서체** — 읽기용 세리프가 아니라 산세리프로 렌더링

## 1.8.2 - 2026-07-02

이번 릴리스의 중심은 **컨테이너·샌드박싱**입니다. 모델이 짠 코드를 격리 실행하는
**"샌드박싱" 개념**을 새로 더하고, 카탈로그의 코드 샌드박스 분류를 **"샌드박스·런타임"**
으로 넓혀 Modal·Daytona·Docker를 추가했으며, 모델이 쓴 파이썬을 일회용 Docker
컨테이너에서 돌리는 **실행 가능한 튜토리얼 한 편**을 새로 실었습니다. 여기에 설명 필드의
인라인 마크다운 렌더링, 개념 인덱스·정렬 정리, 본문 소프트랩, 모바일 링크 동작과 헤딩
여백 손질을 함께 담았습니다.

### 추가됨 1.8.2

- **"샌드박싱" 개념** — 모델이 짠 코드·명령을 격리된 환경에서 실행하는 샌드박싱을 다루는
  개념을 추가하고, 하네스 엔지니어링·"도구" 개념과 서로 링크
- **"샌드박스·런타임" 카테고리 확장** — 기존 "코드 샌드박스" 분류를 "샌드박스·런타임"으로
  넓히고 Modal·Daytona·Docker를 카탈로그에 추가
- **Docker 코드 샌드박스 에이전트 튜토리얼** — 모델이 쓴 파이썬을 네트워크 차단·자원 제한
  일회용 Docker 컨테이너에서 실행하는 실행 가능한 샘플(`samples/docker_1`)과 글을 추가.
  Docker-out-of-Docker(DooD) 실행법과 실제 실행 결과, 히어로 이미지를 함께 실음
- **설명 필드 인라인 마크다운** — 도구·개념·글의 설명에 쓰인 `` `code` ``·굵게·기울임을
  실제 서식으로 렌더링
- **모바일에서 현재 창으로 열기** — 좁은 화면(≤640px)에서는 새 탭으로 열리던 링크를 현재
  창에서 열도록 함. 데스크톱은 새 탭을 유지하고, 창 폭이 넓어지면 원래대로 복원

### 변경됨 1.8.2

- **개념 인덱스 정리** — 카드 그리드를 한 줄 3개로 좁히고(기존 2개는 너무 큼), 기본 정렬을
