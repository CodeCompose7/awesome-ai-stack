## Overview

LiteLLM is a thin abstraction over many model providers (Anthropic, OpenAI,
Bedrock, Vertex, local models, …) exposing a single OpenAI-compatible call.
Beyond the SDK, its **proxy server** acts as an LLM gateway: load balancing,
automatic fallbacks, retries, budgets, and per-key cost tracking — so agents
can swap models without code changes.

## Sample: one interface, any provider

```python
from litellm import completion
import os

os.environ["ANTHROPIC_API_KEY"] = "sk-ant-..."

resp = completion(
    model="anthropic/claude-opus-4-8",
    messages=[{"role": "user", "content": "Summarize litellm in one line."}],
)
print(resp.choices[0].message.content)

# Switch providers by changing only the model string, e.g.:
#   model="openai/gpt-4o"  or  model="bedrock/anthropic.claude-3-5-sonnet"
```

## When to use it

Use LiteLLM when an agent must stay provider-agnostic, needs fallbacks across
models, or when you want a central gateway to enforce budgets and track spend.
