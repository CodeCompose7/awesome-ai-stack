## 개요

LiteLLM은 여러 모델 제공자(Anthropic, OpenAI, Bedrock, Vertex, 로컬 모델 등)
위에 얇은 추상화를 씌워 하나의 OpenAI 호환 호출로 노출합니다. SDK뿐 아니라
**프록시 서버**가 LLM 게이트웨이 역할을 합니다 — 로드 밸런싱, 자동 폴백,
재시도, 예산, 키별 비용 추적까지. 덕분에 에이전트는 코드 변경 없이 모델을
교체할 수 있습니다.

## 샘플: 하나의 인터페이스, 모든 제공자

```python
from litellm import completion
import os

os.environ["ANTHROPIC_API_KEY"] = "sk-ant-..."

resp = completion(
    model="anthropic/claude-opus-4-8",
    messages=[{"role": "user", "content": "Summarize litellm in one line."}],
)
print(resp.choices[0].message.content)

# 모델 문자열만 바꾸면 제공자가 바뀝니다. 예:
#   model="openai/gpt-4o"  또는  model="bedrock/anthropic.claude-3-5-sonnet"
```

## 언제 쓰면 좋은가

에이전트가 제공자에 종속되지 않아야 하거나, 모델 간 폴백이 필요하거나, 예산을
강제하고 지출을 추적할 중앙 게이트웨이가 필요할 때 LiteLLM을 사용하세요.
