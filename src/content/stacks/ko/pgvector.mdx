## 개요

`pgvector`는 PostgreSQL에 `vector` 컬럼 타입과 유사도 연산자를 추가합니다.
이미 Postgres를 운영 중이라면, 에이전트에 장기 기억이나 RAG 검색을 붙이는
가장 마찰 없는 방법입니다 — 추가로 운영할 서비스가 없습니다.

## 샘플: 임베딩 저장과 조회

```sql
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE memories (
  id    bigserial PRIMARY KEY,
  body  text,
  embedding vector(1536)        -- 임베딩 모델의 차원 수에 맞추세요
);

-- 코사인 거리(<=>)를 이용한 최근접 이웃 검색:
SELECT body
FROM memories
ORDER BY embedding <=> $1
LIMIT 5;
```

테이블이 커지면 근사 인덱스(`hnsw` 또는 `ivfflat`)를 추가해 쿼리 속도를
유지하세요:

```sql
CREATE INDEX ON memories USING hnsw (embedding vector_cosine_ops);
```

## 언제 쓰면 좋은가

벡터 검색을 관계형 데이터와 같은 곳에 두고 싶고, 전용 벡터 스토어를 추가하기
보다 하나의 데이터베이스만 운영하고 싶을 때 pgvector를 선택하세요.
